# Техническое руководство по интеграции SIP-телефонии Beeline с SalesRender CRM

## 1. Спецификация проекта

### 1.1. Назначение
Интеграция SIP-телефонии оператора Beeline с CRM-системой SalesRender для автоматизации обработки входящих и исходящих звонков.

### 1.2. Целевые показатели
- Увеличение конверсии лидов в звонки на 50%
- Снижение стоимости привлечения клиента (CPA) на 20%
- Автоматизация обработки входящих лидов

### 1.3. Технические требования
- Аккаунт Beeline Бизнес с подключенной услугой SIP-транк
- SalesRender CRM с правами администратора
- Бюджет: 5000 руб/мес (10 каналов связи)
- Тарифная ставка: ~2 руб/мин

## 2. Процедура настройки

### 2.1. Конфигурация SIP-транка в Beeline

#### 2.1.1. Заказ услуги SIP-транк
1. Авторизуйтесь в Личном кабинете Beeline Бизнес
2. Перейдите в раздел: "Услуги"
3. В разделе "Услуги" выберите пункт "Виртуальная АТС / SIP-транк"
3. Активируйте услугу SIP-транк
4. Настройте параметры конфигурации:
   - Тип номера: многорегиональный (+7-495...) или 8-800
   - Количество каналов: 5-10
   - Режим работы: полнодуплексный

#### 2.1.2. Получение учетных данных
1. Перейдите в раздел "SIP-настройки"
2. Зафиксируйте следующие параметры подключения:

| Параметр | Значение | Протокол |
|----------|----------|----------|
| Сервер | sip.beeline.ru | SIP |
| Порт | 5060 | UDP/TCP |
| Логин | SIP02C000XXXXX | - |
| Пароль | [сгенерированный] | - |
| Domain | sip.beeline.ru | SIP |

#### 2.1.3. Проверка соединения
1. Выполните тестовый звонок на назначенный виртуальный номер
2. Убедитесь в успешной регистрации SIP-транка
3. Зафиксируйте качество связи

### 2.2. Интеграция с SalesRender CRM

#### 2.2.1. Добавление провайдера телефонии
1. Войдите в интерфейс SalesRender с правами администратора
2. Перейти в раздел: Колл-центр
3. В разделе "Колл-центр" выберите "Настройки"
4. В настройках выберите параметр "Провайдеры телефонии"
5. Выберите действие: "Добавить провайдера"
6. Выберите тип провайдера: "Beeline" или "Generic SIP"

#### 2.2.2. Конфигурация параметров подключения
1. Заполните форму параметрами подключения:

| Параметр | Значение | Примечание |
|----------|----------|------------|
| Наименование | Beeline_Main | Идентификатор подключения |
| Сервер | sip.beeline.ru | SIP proxy сервер |
| Порт | 5060 | UDP порт |
| Логин | [логин из ЛК Beeline] | SIP User ID |
| Пароль | [пароль из ЛК Beeline] | Аутентификация |
| CIDR | 0.0.0.0/0 | Маска номеров для привязки |
| Количество каналов | 5-10 | Лимит одновременных соединений |
| Запись разговоров | Активирована | Период хранения: 30 дней |

#### 2.2.3. Валидация подключения
1. Нажмите кнопку "Тестировать" в интерфейсе SalesRender
2. Дождитесь тестового звонка на мобильный телефон
3. Подтвердите успешность установления соединения

### 2.3. Привязка к бизнес-процессам

#### 2.3.1. Настройка проектов и офферов
1. Перейдите в раздел "Оффер"
2. Нажмите на параметр "Редактировать"
3. Выберите пункт "Телефония"
3. Назначьте провайдера: Beeline_Main
4. Укажите номер оффера для CID: +7XXXXXXXXX

#### 2.3.2. Настройка соответствия полей
1. Откройте вкладку "Соответствие полей" в настройках оффера
2. Добавьте соответствие для поля "phone" → #phone
3. Установите видимость телефона: "Частично скрыто"

### 2.4. Процедура тестирования

#### 2.4.1. Тестирование входящих звонков
1. Инициируйте звонок на виртуальный номер Beeline
2. Проверьте создание карточки лида в CRM
3. Убедитесь в наличии записи звонка в карточке лида

#### 2.4.2. Тестирование исходящих звонков
1. Из карточки лида нажмите кнопку "Позвонить"
2. Убедитесь, что у абонента отображается номер оффера
3. Проверьте фиксацию звонка в истории лида

#### 2.4.3. Проверка записи разговоров
1. Прослушайте запись тестового звонка
2. Убедитесь, что длительность записи превышает 60 секунд
3. Проверьте качество аудиозаписи

## 3. Автоматизация и интеграции

### 3.1. Настройка триггеров
1. Перейдите в раздел "Настройки"
2. Выберите пункт "Триггеры"
2. Создайте новый триггер с условием "Новый лид"
3. Настройте действие: "Обзвонить через 5 минут"

### 3.2. Интеграция с лендинг-платформами
1. Настройте callback-функцию на лендинге Tilda
2. Реализуйте POST-запрос к API SalesRender при отправке формы
3. Настройте postback-запросы для трекинга конверсий

### 3.3. API для управления звонками
Используйте следующие конечные точки  для инициации звонков:

curl -X POST https://api.salesrender.com/v1/calls \
  -H "Authorization: Bearer TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"phone": "+79123456789", "offer_id": 123}'

## 4. Мониторинг и аналитика

### 4.1. Настройка отчетности

1. Перейдите в раздел: Колл-центр
2. Выберите раздел "Отчёты" 
3. Выберите параметр Аналитика эффективности
2. Настройте дашборд с ключевыми метриками

### 4.2. Ключевые метрики для мониторинга

| Метрика | Целевое значение | Метод оптимизации |
|---------|------------------|-------------------|
| Соотношение звонков к лидам | >20% | Настройка IVR-меню |
| Средняя длительность разговора | >60 секунд | Оптимизация скриптов |
| Конверсия в целевое действие | >10% | Внедрение распознавания речи |
| Return on Advertising Spend | >3x | Таргетирование через RuTarget |

## 5. Устранение неисправностей

### 5.1. Диагностика проблем с регистрацией SIP

1. Проверьте доступность порта 5060
2. Настройте правила firewall для UDP/TCP трафика
3. Убедитесь в корректности логина и пароля

### 5.2. Решение проблем с привязкой звонков

1. Проверьте настройки CIDR-маски (рекомендуется 0.0.0.0/0 для тестирования)
2. Убедитесь в корректном соответствии полей телефона
3. Проверьте настройки номеров в офферах

### 5.3. Контроль бюджета

1. Установите лимит в 5 каналов на начальном этапе
2. Настройте уведомления о приближении к лимиту
3. Регулярно проверяйте расходы в личном кабинете Beeline

## 6. Дальнейшее развитие системы

### 6.1. Внедрение дополнительного функционала

1. Настройте IVR-меню для автоматической маршрутизации
2. Интегрируйте в  систему распознавания речи
3. Подключите системы сквозной аналитики (UIS/Calltouch)

### 6.2. Масштабирование системы

1. Увеличите количество каналов до 10-20
2. Настроите обработку 100+ лидов в день
3. Реализуйте распределение звонков между операторами

### 6.3. Альтернативные решения

1. Проанализируйте тарифы провайдеров MTC/МегаФон
2. Оцените экономический эффект от перехода на альтернативных операторов
3. Разработайте план миграции при необходимости
